Query a Rust codebase via syntactic (syn) analysis.

Precision tiers (look for the `via` column on results to see which fired):

PRECISE (raw AST shape — trustworthy):
- Item inventory, impl blocks, struct fields, enum variants.
- self.field accesses inside `impl Type`.
- Match-site / pattern shapes (catch-all-arms, parallel-matches, if-chains).
- Free-fn / method / macro call sites by last-segment name.

APPROXIMATE (semantic-lite, may have false positives/negatives):
- field-uses `via=ti`: receiver type inferred from local lets, params,
  and obvious constructors. Misses method-chain results, generics,
  trait dispatch, closure captures.
- type-refs `via=alias`: walks `type X = Y;` chains. Misses associated-
  type re-exports (`impl Foo { type Out = Bar; }`).
- callers `Type::method`: also matches paths where the head segment is
  renamed by `use foo::Type as Other`. Misses dyn-dispatch and generics.
- dead-code: last-segment name matching; pub items may have external
  callers we don't see; trait methods are skipped to cut false positives.

BEST-EFFORT (heuristic — flag and verify):
- field-uses `via=?` (--candidates only): receiver type unknown.
- Macro body scanning: token streams parsed speculatively as expressions.
- Custom macros and DSLs whose args aren't expressions are missed.

Top-level flags:
- --root/-r <path>: root directory (or file) to scan (default `.`);
  respects .gitignore.
- --scope production (default) / tests / all: controls test-code inclusion.
- --cfg KEY[=VALUE]: repeatable. Items whose cfg evaluates to definitively
  False under this env are stripped. Unknown keys leave items in.
- --summary: skip per-row output, keep summary line.

DESIGN AUDIT PLAYBOOK — pick the theme matching your concern.
The tool finds candidates; you decide whether to act.

INDEX (five themes, jump to the one you need):
  1. TYPE & DATA DESIGN     — when types should change shape
  2. ENCAPSULATION & API    — what's leaking, hidden, or expensive to change
  3. DISPATCH & CONTROL FLOW — branching smells: scattered logic, god fns
  4. CORRECTNESS & SAFETY   — silent errors, silent data loss
  5. AUDIT META             — auditing the test suite itself

──── 1. TYPE & DATA DESIGN ──────────────────────────────────────────
When concrete types should become traits, primitives become newtypes,
structs should split, or duplicate concepts should merge.

◇ EXTRACT A TRAIT (concrete type → interface)
  unruster takes-mut <Type>                       # mutation surface
  unruster type-refs <Type>                       # modules naming it
  unruster callers --by module <Type>::<method>   # per-method caller spread
  unruster inventory --kind impl-fn | grep '<Type>::'   # method count
  Signal: many `&mut Type` fns + many naming modules + most callers
  touch only a subset → extract trait(s); callers depend on interface.

◇ NEWTYPE (primitive overloaded across roles — `String`/`u32` as both id
  and value)
  unruster type-refs String                       # often huge
  unruster takes-mut String
  unruster callers <fn-taking-primitive> --by module
  Signal: same primitive returned/accepted across unrelated APIs →
  wrap each role (`UserId(u32)`, `Cents(u64)`) so the compiler catches
  mix-ups.

◇ SPLIT A STRUCT (low-cohesion / god-struct)
  unruster fields <Type>                          # field count
  unruster metrics --top 10                       # top structs by field count
  unruster field-uses <Type> <field> --candidates # field-to-callers map
  Signal: disjoint sets of fns touch disjoint sets of fields → split.

◇ DEAD ENUM VARIANTS
  unruster variants <Enum>
  Signal: 0 ctor sites + only seen in `_ =>` arms → drop the variant.

◇ REDUCE DATA REPLICATION / REPETITION / CONVERSION
  unruster impls --trait From / Into / TryFrom / AsRef
  unruster fields <A> ; unruster fields <B>       # shape overlap
  unruster conversion-pairs                       # mutual From pairs
  unruster pass-through                           # thin wrapper layers
  unruster callers .clone                         # clone hotspots
  unruster callers .to_string / .to_owned / .into # conversion hotspots
  unruster inventory --kind fn | grep '::(from|to|as|into)_'
  Signals: bidirectional `A ↔ B` From + overlapping fields = same
  concept in two shapes (merge); heavy `.clone()` on one type =
  fragmented ownership (Rc/Arc/borrow); sprawling `to_*`/`from_*`
  namespaces on one type = collapse to standard trait impls.

──── 2. ENCAPSULATION & API SURFACE ─────────────────────────────────
What's leaking out, what should be hidden, what's the cost of changing
a published signature.

◇ PRIVATIZE A FIELD (stop external write bleeding)
  unruster fields <Type>                          # pub/priv breakdown
  unruster field-uses <Type> <field> --candidates
  Signal: writes from contexts outside `Type::*` → make field private,
  expose a method that enforces the invariant.

◇ SHRINK A PUB SURFACE (internal API hygiene)
  unruster inventory --vis pub --kind impl-fn
  unruster dead-code --pub-only
  unruster callers --by module <Type>::<method>
  Signal: pub fn with 0 callers → privatize. pub fn called from 1
  module → consider `pub(crate)` or `pub(super)`.

◇ BREAKING-CHANGE BLAST RADIUS (before renaming/changing a signature)
  unruster callers <Type>::<method>
  unruster callers --transitive <Type>::<method>
  unruster type-refs <Type>
  Signal: direct + transitive caller count + coupling footprint tells
  you the change cost.

──── 3. DISPATCH & CONTROL FLOW ─────────────────────────────────────
Branching design smells: scattered per-variant logic, brittle string
dispatch, oversized functions that should split.

◇ REPLACE ENUM-MATCH WITH POLYMORPHISM
  unruster parallel-matches <Enum>                # match sites by variant set
  unruster variants <Enum>                        # ctor + match per variant
  unruster catch-all-arms <Enum>                  # `_ =>` arms (knowledge leak)
  Signal: ≥2 match sites cover the same variant set → push behavior
  into a trait method per variant; callers stop knowing the variants.

◇ PARTIAL-ENUMERATION DEFECT (a predicate that silently mis-bins a NEW
  variant — run BEFORE adding a variant to find every site that won't
  force-update)
  Also run AFTER adding / promoting / splitting a variant, and after any
  "extract variant to its own type" refactor. The pre-refactor sites
  that matched on the old variant set are the candidates: they didn't
  break compilation but may now silently drop the new variant.
  unruster enum-coverage <Enum>                   # the one-stop synthesis
  unruster parallel-matches <Enum> --hide-exhaustive --rank-by-gap --show-missing
  unruster parallel-matches <Enum> --include-matches-macro
  unruster parallel-matches <Enum> --include-if-chains
  Signal: `enum-coverage` lists every PARTIAL match / `matches!` /
  `==`-if-chain site — one row per (gap_score, covered, missing,
  file:line), sorted by gap_score = covered/total descending. The top
  rows are predicates covering almost every variant (e.g. 7/8): a new
  variant falls into their `_` / `matches!`-false / dispatch-`else` arm
  with zero compiler warning. Exhaustive sites are compiler-protected
  and hidden. `matches!()` is guaranteed-supported (always on in
  enum-coverage; opt-in elsewhere via --include-matches-macro).
  `if x == E::A { … } else if x == E::B …` chains are scanned the same
  way (always on in enum-coverage; opt-in elsewhere via
  --include-if-chains) — their implicit `else` IS the risk that `match`
  would force you to handle.
  
  Strongest smell — ASYMMETRIC DIVERGENCE between sibling sites. When
  two sites with similar names / intent (e.g. `as_paintable`,
  `paint_slots`, `collect_world_transforms`) have overlapping but
  non-equal coverage sets, the smaller one is almost always the bug:
  both meant the same conceptual filter, one drifted when a variant was
  added. Pick out divergent-coverage clusters with:
    unruster -r src enum-coverage <Enum> | sort -k4 | uniq -f3 -D
  (sorts on the covered-set column to group near-identical predicates).
  
  False positives to skip — TRAIT-ROUTED CATCH-ALLS. A row whose `_` /
  catchall arm calls a no-default trait classifier on the scrutinee
  (`node.kind()`, `node.as_paintable()`, `node.capabilities()`) is
  structurally safe: a new variant must implement the trait method, so
  the catch-all picks up its behavior automatically. The tool can't see
  through the method call, so it tags such rows
  `(catchall→method; likely false positive)` and
  `--hide-trait-routed-catchalls` drops them entirely. Read the `_` arm.
  
  Two repair shapes:
  - Trait-classifier replacement (preferred). Add a no-default method on
    the enum's primary trait (`fn classification(&self) -> Class;`).
    Every existing variant must declare it; the buggy site routes through
    it; future variants can't compile until they declare it. A partial
    `matches!(node, BaseShape | CompositeShape | ConstraintDerivedShape)`
    becomes `node.paintable_kind() == PaintableKind::Path`.
  - Exhaustive match. If the dispatch genuinely differs per variant,
    replace `matches!(.., A | B | C)` with a full
    `match { A => .., B => .., C => .., D | E | F => .. }` — exhaustive,
    so a new variant forces a compile error at this site.

◇ SIBLING-COHORT DIVERGENCE (one sibling in a `do_x_*` family forgot a
  call its cohort-mates all make — e.g. `wrap_in_transform` skipped
  `mark_pending` that `wrap_in_group`/`_composite` both call)
  unruster callers <helper> --among '<name-glob>'  # who calls it / who doesn't
  unruster cohort-callees '<name-glob>'             # (callee × fn) matrix
  Signal: `--among` lists each cohort fn ✓ (calls helper) or ✗ (doesn't);
  the ✗ rows are candidates. `cohort-callees` shows the whole grid —
  a callee present in the majority of columns but missing from one is
  flagged `<- divergence`. The tool finds the asymmetry; you decide
  whether that sibling SHOULD have made the call (some omissions are
  correct — not every sibling wraps an expandable kind).

◇ CO-CALL INVARIANT (paired actions — calling one without the other
  leaks an invariant; e.g. `refresh_world_transforms` +
  `recompute_derived_geometry` both must run to settle a `Document`,
  so any fn that calls one alone may leave stale state behind)
  unruster co-call <A> <B>                          # asymmetric callers
  When to reach for it: you know two ops form a couple (mutate +
  reindex, lock + unlock, begin + commit, invalidate + recompute) and
  want every site that does one but forgot the other. Unlike
  `--among`/`cohort-callees` (which need a NAME cohort like `wrap_in_*`),
  co-call needs no naming convention — it partitions ALL fns by the two
  calls. A and B take the same target forms as `callers` (bare name,
  `Type::method`, `.method`, `::name`, `name!`).
  Output: two row kinds, ranked by matched-call count (high-traffic
  first), each with a `via file:line` pointer to the call it DOES make:
    A-only <n> <fn> via f:L   # calls A, not B — forgot B?
    B-only <n> <fn> via f:L   # calls B, not A — redundant A, or B alone
  (grep `^A-only` / `^B-only` to split the streams). The stderr summary
  counts the canonical both-callers, which are not listed.
  Signal: each row is a latent stale-state candidate. Some asymmetries
  are correct and you dismiss them — e.g. a typed gate that only QUEUES
  a mutation (the later `commit` runs B), an orchestrator that dispatches
  to actions that run B internally, or a solver path where the omission
  is intentional. Expect a minority of true positives (the tool surfaces
  candidates; you filter) — same working style as `enum-coverage`.
  Tip: one pair only checks one invariant. Re-run with a different pair
  to police a different couple (e.g. `co-call invalidate_cache
  recompute_derived_geometry` for cache sites that skip the recompute).

◇ STRINGLY-TYPED CODE (logic branches on string literals)
  unruster stringly                               # ==, .eq, match, assert_eq!
  unruster stringly --include-substring           # also .starts_with/.contains
  unruster stringly --include-map-keys            # also map.get("lit")
  unruster stringly --by fn                       # rank worst offenders
  Signals: multiple `match-lit` arms in one fn → replace with enum
  (compiler catches typos); `cmp-eq`/`cmp-method` hits → newtype the id
  (e.g. `pub struct ActionId(&'static str)`); `map-lit-key` →
  `HashMap<MyEnumKey, V>` instead of `HashMap<String, V>`.

◇ GOD FUNCTION TO SPLIT (long / complex / deeply-nested)
  unruster metrics --sort loc --top 20            # by line count
  unruster metrics --sort cyclo --top 20          # by cyclomatic complexity
  unruster metrics --sort nesting --top 20        # by max nesting depth
  unruster metrics --sort cyclo --threshold 15    # above-threshold
  unruster metrics --sort nesting --threshold 4   # deeply-indented
  unruster callees <Type>::<fn>                   # helper-call clustering
  Signals: high LOC + low cyclo → extract named sections, not split;
  high cyclo (≥15) → split decision logic into focused helpers, or
  trait-dispatch (theme 3); high nesting (≥4) → flatten with early
  returns / guard clauses; all three high → genuine god fn, split first.

──── 4. CORRECTNESS & SAFETY ────────────────────────────────────────
Runtime risks hidden in the code: silent error-swallowing, silent data
loss from casts and conversions.

◇ SILENT FALLBACKS (Result/Option error swallowing)
  unruster error-swallows [--include-unwrap-or]
  Signal: `match-err-wild` / `if-let-ok` / `let-_` / `.map_err(|_|)` /
  `.unwrap_or_default()` hits. Some are intentional (parse cascades,
  drop guards) — review per site.

◇ EXCESSIVE CASTS / SHAPE-JUGGLING (the same value renamed/reshaped
  repeatedly because the wrong type was chosen at the boundary)
  unruster casts                                  # all `as` casts, classified
  unruster casts --class narrow-int,signed-flip,float-int,ptr
  unruster casts --by fn                          # rank cast-heavy fns
  unruster casts --hide-widen                     # hide safe widenings
  unruster conversions --by fn --top 10           # conversion-heavy fns
  unruster conversions --kind .into,.to_string,::from
  Signals: one fn doing many casts → wrong-typed input, cast once at
  boundary; `narrow-int`/`signed-flip`/`float-int` → silent data-loss
  candidates (prove safe or use checked helper); fn with 5+ conversion
  calls = surrounding API wants the wrong type.

──── 5. AUDIT META ─────────────────────────────────────────────────
Auditing the test suite itself — coverage imbalance, near-duplicates,
missing flag combos.

◇ TEST-SUITE AUDIT
  unruster tests                                  # list #[test] fns + file:start-end
  unruster tests --with-hint                      # add args() fingerprint per test
  unruster tests --by subcommand                  # histogram by CLI subcommand
  Signals: `--by subcommand` exposes coverage imbalance (5 tests vs 1);
  `--with-hint` exposes near-duplicates (same args fingerprint = same
  test in disguise); `file:start-end` lets an agent read just the
  relevant body via `sed -n start,endp file`.

─────────────────────────────────────────────────────────────────────
Closing principle: the tool finds candidates. The decision "extract a
trait" / "newtype that primitive" / "split that fn" / "merge these
two types" stays with the person reading the output.
